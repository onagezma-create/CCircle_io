# Платежи и товары в Яндекс.Играх (Circle.io)

Краткий гайд, как подключить **внутриигровые покупки**, чтобы игроки могли платить за скины и премиум-карты.

## 1. Требования

- Игра **опубликована** на [Яндекс.Играх](https://yandex.ru/dev/games/) (черновик с ограничениями может не выдавать реальные платежи).
- В коде используется SDK `YaGames` и методы `getPayments()` / `payments.purchase()`.

## 2. Создать товары в кабинете разработчика

1. Откройте [Консоль Яндекс.Игр](https://developer.tech.yandex.ru/) → ваш проект → раздел про **монетизацию** / **внутриигровые покупки** (название может быть «Товары», «Покупки», «In-app»).
2. Создайте товар для каждого **строкового ID**, который зашит в игре:
   - Скины: `circle_skin_ruby`, `circle_skin_sapphire` (и другие, если добавите).
   - Премиум-карта: `circle_map_crystal29` (цена в кабинете — например **29 ₽**).
3. Тип: обычно **внутриигровая покупка** (consumable/non-consumable — по правилам площадки для вашего случая).
4. Сохраните и **дождитесь модерации** товаров, если она требуется.

ID в кабинете и в коде должны совпадать **символ в символ** с полями `yandexProductId` в `index.html` (объекты `SKINS` и `COUNTRIES`).

## 3. Как это вызывается в игре

- Скины с `unlock: 'purchase'` открываются через `purchaseSkinYandex(skinId)`.
- Карта **«Кристальный рай»** (id `7`) — через `purchaseCountryYandex(7)`; товар: `circle_map_crystal29`.
- После успешной покупки прогресс сохраняется в `localStorage` и синхронизируется в Player API (`syncYaPlayerProgress`), если SDK доступен.
- При старте вызывается `restoreYandexPurchases()` — подтягивает уже купленные товары из каталога Яндекса.

## 4. Реклама за награду (карты «Ромашковая долина», «Империя бургеров»)

- В консоли должна быть включена **монетизация с рекламой**; для rewarded нужен доступ к **rewarded video** в SDK (`sdk.adv.showRewardedVideo`).
- Локально (`localhost`) для теста разблокировка может сработать без рекламы (см. логику `unlockCountryFromAd`).

## 5. Типичные проблемы

| Симптом | Что проверить |
|--------|----------------|
| `SDK не загружен` | Игра открыта не на домене Яндекса / SDK не подключён. |
| Покупка не открывается | Товар не создан или ID не совпадает с `yandexProductId`. |
| Ошибка после оплаты | Логи консоли, версия SDK, что покупка есть в `getPurchases()`. |

## 6. Документация Яндекса

Актуальные разделы ищите на официальном сайте: **Яндекс Игры → SDK → Платежи и покупки** (URL может обновляться; ориентируйтесь на раздел «Payments» / «Внутриигровые покупки» в [документации](https://yandex.ru/dev/games/doc/)).

## 7. Отклонение: «URL внутреннего хранилища» / сторонняя БД

Робот модерации часто ругается на:

- строку **`https://sdk.games.s3.yandex.net/...`** в коде (считается «внутренним» хранилищем площадки) — в проекте SDK подключается только как **`/sdk.js`**;
- **Firebase** (`gstatic.com/firebase`, `firebasedatabase.app`) — в **`index.html` для архива Яндекса этих URL больше нет**; они только в файле **`firebase-cloud.js`**.

**Что класть в ZIP для Яндекс.Игр:** `index.html`, папка **`assets/`**, при необходимости иконки — **без** `firebase-cloud.js` (на домене Яндекса облако Firebase не подключается, сохранения — локально + Player API).

Для **своего сайта / localhost с входом по почте** положите рядом **`firebase-cloud.js`** (как сейчас в репозитории).

Если снова отклонят из‑за **Railway** (`ccircleio-production.up.railway.app`) — в комментарии модератору опишите внешний игровой сервер мультиплеера или временно отключите MP только для проверки (обсуждайте с поддержкой площадки).
